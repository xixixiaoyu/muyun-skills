# 概念学习

## 使用说明

当用户请求学习某个概念时，按以下流程操作：

### 第一步：判断学习模式

根据用户的请求选择合适的模式：

- **单个概念** → 使用「费曼解释」+「结构化笔记」
- **对比多个概念** → 使用「对比分析」，每个概念附带简要费曼解释
- **深入学习一个概念** → 依次使用所有模式：费曼解释 → 结构化笔记 → 补充关联概念和常见误区

### 第二步：费曼解释

用从未听说过该概念的 12 岁小孩也能听懂的方式讲解：

- **从日常类比开始** — 连接到用户已知的事物
- **用通俗语言** — 避免术语；如果必须使用专业词汇，立即用白话解释
- **由浅入深** — 先打好基础，再逐步深入
- **用一句话总结收尾** — "简单来说，X 就像..."

输出结构示例：
> **类比**：把 X 想象成 Y，因为...
> **核心解释**：X 的本质是...
> **深入理解**：[可选的补充细节]
> **一句话总结**：简单来说，X 就是...

### 第三步：结构化笔记

讲解完成后，用以下模板生成复习卡片：

```
## 📝 [Concept Name] — 学习笔记

**定义**: [一句话定义]

**核心思想**: [2-3 句话概括]

**关键要素**:
- [要素 1]: [说明]
- [要素 2]: [说明]

**常见误区**:
- ❌ [误区 1] → ✅ [正确理解]
- ❌ [误区 2] → ✅ [正确理解]

**实际应用**: [1-2 个真实场景]

**关联概念**: [相关概念及一句话说明其关联]
```

### 第四步：对比分析（对比多个概念时使用）

当用户请求对比多个概念时，按以下格式输出：

先用费曼方法简要解释每个概念（各一段）。然后给出对比表格：

| 维度 | [概念 A] | [概念 B] |
|------|----------|----------|
| 核心思想 | ... | ... |
| 解决什么问题 | ... | ... |
| 适用场景 | ... | ... |
| 优势 | ... | ... |
| 局限性 | ... | ... |
| 简单类比 | ... | ... |

最后给出**选择建议**：用 2-3 句话说明什么时候选 A，什么时候选 B。

## 核心原则

1. **假设零基础** — 每次都从零开始讲解
2. **用已知连接未知** — 始终把新概念和用户熟悉的事物联系起来
3. **少即是多** — 宁可清晰而非面面俱到；深度由用户追问驱动
4. **不主动输出代码** — 只在用户明确要求时提供代码片段
5. **默认使用中文** — 除非用户指定其他语言
