# 代码移除与迭代计划模板

## 优先级

- [ ] **P0**：需要立即移除（安全风险、显著成本、阻塞其他工作）
- [ ] **P1**：当前迭代内移除
- [ ] **P2**：待办/下次迭代

---

## 可立即安全移除

### 项目：[名称/描述]

| 字段         | 详情                                        |
| ------------ | ------------------------------------------- |
| **位置**     | `path/to/file.ts:line`                      |
| **原因**     | 为什么应该移除                              |
| **证据**     | 未使用（无引用）、死 Feature Flag、废弃 API |
| **影响**     | 无 / 低 - 无活跃消费者                      |
| **删除步骤** | 1. 移除代码 2. 移除测试 3. 移除配置         |
| **验证**     | 运行测试、检查无运行时错误、监控日志        |

### 示例：未使用的工具函数

| 字段         | 详情                                        |
| ------------ | ------------------------------------------- |
| **位置**     | `src/utils/deprecated.ts:15-45`             |
| **原因**     | 函数 `formatOldDate` 在重构后不再被调用     |
| **证据**     | `rg "formatOldDate"` 仅返回定义位置，无调用 |
| **影响**     | 无 - 纯工具函数，无副作用                   |
| **删除步骤** | 1. 删除函数定义 2. 删除相关测试 3. 更新导出 |
| **验证**     | `npm test` 通过，构建成功                   |

## 延迟移除（需要计划）

### 项目：[名称/描述]

| 字段           | 详情                                       |
| -------------- | ------------------------------------------ |
| **位置**       | `path/to/file.ts:line`                     |
| **延迟原因**   | 有活跃消费者、需要迁移、需要利益相关者签字 |
| **前置条件**   | Feature Flag 关闭 2 周、遥测显示 0 使用量  |
| **破坏性变更** | 列出任何 API/契约变更                      |
| **迁移计划**   | 消费者迁移步骤                             |
| **时间线**     | 目标日期或迭代                             |
| **负责人**     | 负责人/团队                                |
| **验证**       | 确认安全移除的指标（错误率、使用量计数）   |
| **回滚计划**   | 发现问题时如何恢复                         |

### 示例：废弃的 API 端点

| 字段           | 详情                                                  |
| -------------- | ----------------------------------------------------- |
| **位置**       | `src/api/v1/legacy-order.ts`                          |
| **延迟原因**   | 仍有 3 个外部客户端在使用                             |
| **前置条件**   | 1. 通知所有客户端 2. 提供迁移文档 3. 监控使用量降至 0 |
| **破坏性变更** | `/api/v1/order` 将返回 410 Gone                       |
| **迁移计划**   | 客户端迁移至 `/api/v2/order`，参数格式变更见文档      |
| **时间线**     | 2024-Q2 末                                            |
| **负责人**     | @backend-team                                         |
| **验证**       | API 调用量监控、错误率监控                            |
| **回滚计划**   | 保留代码 30 天，可快速恢复                            |

## 移除前检查清单

- [ ] 搜索代码库中的所有引用（`rg`、`grep`）
- [ ] 检查动态/反射式使用
- [ ] 验证无外部消费者（API、SDK、文档）
- [ ] 审查 Feature Flag 遥测（如适用）
- [ ] 更新/移除测试
- [ ] 更新文档
- [ ] 通知团队（如果是共享代码）

### 前端额外检查

- [ ] 检查埋点配置是否引用该代码
- [ ] 检查 A/B 测试配置
- [ ] 检查灰度发布配置
- [ ] 检查是否有对应的移动端/桌面端代码需要同步移除

---

## 移除后验证

### 自动化验证

```bash
# 1. 类型检查
npm run type-check

# 2. 单元测试
npm test

# 3. 构建验证
npm run build

# 4. 搜索残留引用
rg "移除的函数名/组件名" --type ts --type js
```

### 手动验证

- [ ] 本地运行应用，检查控制台无报错
- [ ] 测试相关功能流程
- [ ] 检查页面渲染正常

### 监控验证（上线后）

- [ ] 错误监控无新增异常
- [ ] 性能指标无明显下降
- [ ] 用户反馈渠道无相关问题
