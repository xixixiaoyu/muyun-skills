# Docker / CI 基础设施检查清单

覆盖 Docker 容器安全和 CI/CD 流水线安全的通用检测信号。检测到 Dockerfile 或 CI 配置文件时加载。

## Docker 安全

### 运行时安全

- 容器以 root 用户运行 → **P1**，应创建非 root 用户并 `USER` 切换
- 容器使用 `--privileged` 模式或挂载 Docker socket → **P1**
- 敏感环境变量在 `docker run` 命令行明文传递（可被 `ps` 查看）→ **P1**
- 未限制容器资源（CPU/内存），单个容器耗尽宿主机资源 → **P2**

### 镜像构建

- 镜像中包含构建工具（gcc/git/make）、密钥或凭证文件 → **P0**
- 基础镜像未固定版本（使用 `latest` 或 `:1` 浮动标签）→ **P2**
- 未使用多阶段构建（multi-stage build）导致镜像包含编译依赖和中间产物 → **P3**
- `COPY . .` 全量拷贝前未使用 `.dockerignore` 排除 node_modules/.git 等 → **P2**
- 镜像层过多（每个 RUN/COPY 产生一层）未合并优化 → **P3**

### 健康检查与启动

- 未定义 `HEALTHCHECK` 指令 → **P2**（容器运行状态不可知）
- `ENTRYPOINT`/`CMD` 中使用 `sh -c` 而非直接执行（额外 shell 进程，信号传递受阻）→ **P3**
- `EXPOSE` 端口与 `docker run -p` 不一致 → **P3**

### 镜像安全

- 从不可信镜像仓库拉取镜像 → **P1**
- 未对基础镜像做漏洞扫描 → **P2**
- `apt-get install` 等包管理器未清理缓存（`rm -rf /var/lib/apt/lists/*`）→ **P3**

## CI/CD 安全

### 密钥管理

- CI 流水线中密钥通过环境变量明文传递（日志中可打印）→ **P1**
- 密钥通过代码仓库提交（而非 CI 平台 secrets 管理）→ **P0**
- CI 配置文件中硬编码凭据 → **P0**
- secrets 未限制分支范围（所有分支可访问生产密钥）→ **P1**

### 依赖安全

- 依赖完整性校验缺失（未提交 lock 文件、未使用 hash 校验）→ **P1**
- 依赖安装时未固定版本（`npm install` 而非 `npm ci`、`pip install` 无 `--require-hashes`）→ **P2**
- 未对第三方依赖做已知漏洞扫描（`npm audit`/`pip-audit`/`trivy`）→ **P2**

### 流水线安全

- 制品未签名/未生成校验和 → **P2**（无法验证部署制品完整性）
- 流水线脚本从外部未审核的源拉取执行 → **P1**
- 构建缓存未隔离（不同分支共享缓存可能导致污染）→ **P2**
- 部署密钥未设置过期时间 → **P1**

### 发布与回滚

- 无回滚机制（如蓝绿部署/滚动更新的灰度策略缺失）→ **P1**
- 数据库迁移脚本与发布解耦但顺序依赖未显式管理 → **P1**
- 生产环境健康检查/监控未在部署后自动验证 → **P2**
