# 后端基础设施检查清单

覆盖数据库、API 设计、日志、缓存四大后端基础设施维度的通用检测信号。后端项目始终加载。

## 快速索引

| 问题类型 | 级别 | 参考章节 |
|---------|------|---------|
| 连接池无最大连接数/超时 | P1 | 数据库 |
| N+1 ORM 查询 | P1 | 数据库 |
| 缺少必要索引 | P2 | 数据库 |
| 事务隔离级别不当 | P1 | 数据库 |
| 长事务持有锁 | P1 | 数据库 |
| 迁移脚本不可回滚 | P2 | 数据库 |
| 状态变更操作无幂等性 | P1 | API 设计 |

| 列表接口无分页 | P2 | API 设计 |
| 日志打印密码/Token/身份证号 | P0 | 日志 |
| 缓存键无过期时间 | P1 | 缓存 |
| 缓存穿透/雪崩/双写不一致 | P1 | 缓存 |

## 数据库

### 连接池

- 连接池无最大连接数限制 → **P1**（流量突增时耗尽数据库连接）
- 无连接获取超时配置（默认无限等待）→ **P1**
- 无连接泄露检测（未归还的连接长时间持有）→ **P1**
- 连接验证/心跳缺失（静默断开后使用失效连接）→ **P2**

### ORM 查询

- N+1 查询：循环中访问关联对象触发多次数据库查询 → **P1**
- 批量操作使用循环单条 `save` 而非 `saveAll`/`bulkInsert` → **P2**
- 查询返回所有列（`SELECT *`）但仅用少数字段 → **P3**
- 未使用只读事务/查询优化（如 Spring `@Transactional(readOnly=true)`）→ **P3**

### 索引

- 高频查询条件（WHERE/JOIN/ORDER BY）列缺少索引 → **P2**
- 联合索引列顺序不当（最左前缀未匹配查询条件）→ **P2**
- 索引过多导致写入性能下降未评估 → **P2**
- 唯一约束缺失导致业务允许重复数据 → **P1**

### 事务

- 事务隔离级别不匹配业务需求（如默认 READ_COMMITTED 需 REPEATABLE_READ）→ **P1**
- 长事务持有行锁/表锁（事务内包含外部 API 调用/消息发送）→ **P1**
- 部分写入无事务保护（多表更新只部分成功）→ **P1**
- 事务超时配置缺失 → **P2**

### 迁移

- 迁移脚本不可回滚（缺少 down/rollback 步骤）→ **P2**
- 迁移脚本未测试在生产数据量下的执行时间 → **P2**
- 大表 ALTER/加索引无在线迁移方案（导致锁表）→ **P2**
- Schema 变更与代码部署不同步 → **P1**

## API 设计

### 幂等性与安全

- 支付/扣款/创建订单等状态变更操作无幂等键（重试导致重复操作）→ **P1**
- 登录/注册/发送验证码缺少速率限制 → **P1**
- 敏感操作无二次确认/验证码 → **P1**

### 分页与性能

- 列表接口无分页（全量返回可能 OOM）→ **P2**
- 分页使用 offset/limit 而非游标（大偏移量性能差）→ **P2**
- 无请求体大小限制 → **P1**
- 无响应超时配置 → **P2**

### 版本与错误

- API 无版本化策略（破坏性变更影响所有客户端）→ **P2**
- 错误响应格式不统一（有时 JSON、有时 HTML、有时纯文本）→ **P3**
- 错误信息向客户端泄露内部实现细节/堆栈 → **P1**
- HTTP 状态码使用不当（如 200 返回错误信息）→ **P2**

### 输入校验

- 请求参数未在服务端校验（仅依赖前端校验）→ **P1**
- 文件上传无类型/大小/数量限制 → **P1**
- 未对请求参数做类型/范围/长度校验 → **P2**

## 日志

### 敏感数据

- 日志中直接打印密码/Token/API Key/身份证号/手机号 → **P0**
- Cookie/Authorization Header 完整打印到日志 → **P0**
- 异常堆栈中含数据库连接串/密钥信息 → **P1**

### 日志质量

- 异常日志无业务上下文（用户 ID/请求 ID/操作类型）→ **P2**
- 日志级别使用不当：全部 INFO 无 DEBUG/ERROR/WARN 区分 → **P3**
- 生产环境输出 DEBUG 级别日志包含大量敏感调试信息 → **P1**
- 关键业务流程无日志埋点（支付、登录、权限变更）→ **P2**

### 日志框架

- 使用 `console.log`/`System.out.println` 替代结构化日志框架 → **P3**
- 日志配置异步写入但缓冲区满时丢失日志 → **P2**
- 日志文件无轮转策略导致磁盘写满 → **P1**

## 缓存

### 过期与淘汰

- 缓存键无过期时间（TTL），导致内存无限增长 → **P1**
- 热点数据使用相同的过期时间（同时过期引发缓存雪崩）→ **P1**
- 缓存淘汰策略与业务不匹配（如 LRU 不适用需要 TTL 的场景）→ **P3**

### 一致性

- 缓存与数据库双写不一致（先更新 DB 后删缓存 vs 先删缓存后更新 DB 顺序错误）→ **P1**
- 缓存更新失败后无重试/补偿机制 → **P2**
- 多级缓存（本地 + 分布式）数据一致性问题 → **P1**

### 穿透与击穿

- 查询不存在的数据未缓存空值（每次穿透到 DB）→ **P1**
- 热点缓存过期瞬间大量请求击穿到 DB 无互斥锁保护 → **P1**
- 缓存预热策略缺失导致冷启动时全量穿透 → **P2**
